## Design Context

### Users
People who already own raw DNA data from providers like 23andMe, AncestryDNA, MyHeritage, or FTDNA. They range from health-curious consumers to biohackers to people simply wondering "what do my genes say about X?" They arrive with a specific file to upload and a trait question in mind. The job-to-be-done: upload raw data, ask about a trait, and receive a clear, research-backed answer they can trust and understand — all in under a minute.

### Brand Personality
**Curious, Bold, Transparent** — the product invites exploration ("what will you discover?"), makes confident claims backed by real data, and never hides its reasoning. Every finding shows the SNPs, evidence tiers, and citations behind it.

### Emotional Goals
**Discovery & wonder** — the interface should evoke curiosity, excitement, and a sense of awe about the personal nature of genetic data. Users should feel like they're uncovering something meaningful about themselves, not just reading a clinical report.

### Aesthetic Direction
**Warm scientific** — rich gradients, organic shapes, approachable science. Think Apple Health meets Nature journal. The design should feel like a premium research tool that's also welcoming and human. Dark-mode-first (hardcoded `className="dark"` on `<html>`).

**Anti-references**: Must NOT look like a generic SaaS template. Avoid cookie-cutter startup aesthetics, bland card grids, and predictable hero layouts. Every section should feel intentional and distinctive.

**Current palette** (to evolve, not abandon):
- Primary: Teal `#2DD4BF` (dark) / `#0D9488` (light)
- Accent: Amber `#FBBF24` (dark) / `#F59E0B` (light)
- Surfaces: Deep navy-blacks `#0B0B14` → `#12121F` → `#16162A`

**Typography**:
- Display: Space Grotesk (500, 700) — `--font-display`
- Body: IBM Plex Sans (400, 500, 600) — `--font-sans`
- Data/Mono: IBM Plex Mono (400, 500) — `--font-mono`

### Design Principles

1. **Science you can feel** — Data visualization, evidence tiers, and SNP tables should feel alive and meaningful, not sterile. Use motion, color, and hierarchy to make scientific data emotionally engaging.

2. **Earned trust** — Privacy and transparency aren't just features, they're the product's foundation. Design should reinforce trust at every step: clear data handling messages, visible evidence sourcing, honest confidence levels.

3. **Warmth in the dark** — The dark theme should feel warm and inviting, not cold or industrial. Use gradient glows, subtle organic textures, and rich color to create depth and atmosphere.

4. **Progressive revelation** — Don't overwhelm. Lead users from curiosity (hero) → understanding (how it works) → confidence (sample result) → action (upload). Each section earns the next scroll.

5. **Distinctive, not decorative** — Every visual choice should serve comprehension or emotion. Avoid ornamentation for its own sake, but don't be afraid of bold, memorable design moments that make the product unforgettable.

### Tech Stack
- Next.js 16 App Router + React 19
- Tailwind CSS v4 with CSS custom properties (`:root` / `.dark` token system)
- shadcn/ui components (in `src/components/ui/` — leave as-is, redesign consumers)
- Lucide React icons
- `tw-animate-css` for animation utilities
- No new dependencies unless a skill specifically requires one

### Key Constraints
- **Dark-mode-first**: `className="dark"` hardcoded on `<html>` in `layout.tsx`
- **Preserve print styles**: `@media print` block in `globals.css` must not be modified
- **Preserve all functionality**: component props, API calls, auth flows, SSE streaming
- **shadcn/ui**: base components in `src/components/ui/` are untouched — only restyle the custom components and pages that use them
